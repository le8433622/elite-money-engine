# Operations

## Core operating needs
- staging and production environments
- deployment checklist
- error logging
- support intake process
- release notes tracking

## Recommended routines
### Daily
- check health endpoint
- review deployment status
- review critical alerts

### Weekly
- review user feedback
- review failed requests and bugs
- review top feature usage
- prioritize fixes and improvements

### Monthly
- review retention and conversion metrics
- review infrastructure cost
- review product roadmap changes

## Support flow
1. collect issue from user
2. classify as bug, billing, onboarding, or product feedback
3. assign severity
4. resolve and document outcome

## Minimum tooling
- GitHub for source and issues
- deployment platform dashboard
- error tracking
- analytics dashboard
- email support inbox
